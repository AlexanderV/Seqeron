- [[prokaryotic-gene-prediction-rbs]] — annotation-layer prokaryotic ORF-based gene prediction + Shine-Dalgarno RBS detection (ANNOT-GENE-001, `GenomeAnnotator.PredictGenes` / `FindRibosomeBindingSitesBothStrands` / `ScanStrandForShineDalgarno`): both-strand start→stop CDS finding with `protein_length` excluding the stop, and SD consensus `AGGAGG` (+ substrings) matched upstream of a start at an aligned spacing ∈ [minDistance,maxDistance] (score = motifLen/6). Core correctness = the reverse-strand SD mapping `forwardPosition = len − hit.position − motifLen` (scan the reverse complement, since the SD is an mRNA feature); anti-SD 3' 16S tail `YACCUCCUUA`, ~8 nt location, 5 nt literature optimum (Chen 1994). `alternative_to` the Analysis-layer six-frame [[open-reading-frame-detection]] (not contract-equivalent); the strand/spacing-aware counterpart to the bare `AGGAGG` row in [[regulatory-element-detection]]. Validation report [[annot-gene-001-report]].
- [[relative-synonymous-codon-usage]] — RSCU (Sharp & Li 1986) codon-usage-bias measure: n_i·x/Σx normalization, 1.0=no bias, bounded [0,n_i], Σ-over-family=n_i invariant; pooled counts, sense codons only; base anchor of the codon-usage family (vs CAI's max-normalization).
- [[codon-adaptation-index]] — CAI (Sharp & Li 1987) whole-gene codon-bias / expression-proxy score in [0,1]: geometric mean of relative adaptiveness w=f/max_synonym_f, low-value-sensitive, stop excluded; dual include/exclude-Met/Trp mode (exclude can give L=0→0); E. coli oracles; Seqeron deviation = 1e-6 zero-freq clamp (vs Sharp & Li's 0.5 reference-table pseudocount); one level above RSCU.
- [[rare-codon-analysis]] — codon-usage family's *localization/diagnostic* unit (`FindRareCodons` + cluster addendum): a codon is rare when its per-family frequency < threshold (default 0.15), reports position/AA/frequency (E. coli AGA 0.04/AGG 0.02/CGA 0.06/CUA 0.04); addendum adds two published cluster methods — Clarke & Clark 2008 %MinMax (18-codon window, signed %Min/%Max ∈[−100,+100], rare clusters = negative peaks) and Chartier 2012 Sherlocc (7-codon window, ≥4 of 7 freq≤threshold "slow" positions = a rare-codon cluster); single-codon-AA contributes 0 to %MinMax (no NaN), overlapping windows merged; feeds [[codon-optimization]]'s AvoidRareCodons; oracles −86.36/+100/+36.47% and 7×AGA→1 cluster; deviations None.
- [[codon-usage-comparison]] — codon-usage family's *raw* end (`CalculateCodonUsage` + `CompareCodonUsage`): unnormalized codon-count table (non-overlapping triplets, T→U, drop partial codon, Dictionary<codon,int>, Σcounts=total invariant) + Total Variation Distance similarity `1−Σ|f₁−f₂|/2 ∈ [0,1]` between two codon-frequency distributions; proven identity 1.0 / symmetry / range [0,1] / disjoint→0 / partial-overlap-exact properties; empty→0, TVD-not-cosine deliberate metric choice; Kazusa-verified organism tables; shares RSCU's counting primitive, the distribution-comparison op none of the bias measures provide.
- [[codon-optimization]] — codon-usage family's CDS-*rewriting* operation (`OptimizeSequence`): synonymous substitution to improve host expression; five strategies (MaximizeCAI / BalancedOptimization 40-60% GC / HarmonizeExpression=Mignon 2018 / AvoidRareCodons / MinimizeSecondary→ReduceSecondaryStructure); invariants protein-preservation + Met/AUG & Trp/UGG fixed + stop preserved + CAI∈(0,1]; RNA notation, T→U, trim-to-codon, 1e-6 CAI clamp; E. coli/yeast/human preferred-codon fixtures; actuator to RSCU/CAI/ENC's measurement.
- [[effective-number-of-codons]] — ENC/Nc (Wright 1990) reference-free whole-gene codon-bias measure in [20,61]: reciprocal of codon homozygosity F̂=(n·Σp²−1)/(n−1) aggregated by degeneracy class (Nc=2+9/F̂₂+1/F̂₃+5/F̂₄+3/F̂₆); 20=max bias / 61=uniform, unbiased→61, no-bias-sim→40.5; Ile absent→(F̂₂+F̂₄)/2 fallback, overshoot re-adjusts to 61; reference-free counterpart of CAI; one assumption = non-source-prescribed lower clamp at 20.
- [[codon-usage-statistics]] — the codon-usage family's *aggregation/reporting* view (CODON-STATS-001, `CodonUsageAnalyzer.GetStatistics` + `CalculateCai`): one pass bundles CodonCounts, RSCU, ENC, TotalCodons and a **positional-GC block** into a `CodonUsageStatistics` record. Owns what no other codon concept covers — **GC1/GC2/GC3** (% G/C at codon position 1/2/3, EMBOSS `cusp`), **GC3s** (% G/C at *synonymous* third positions, Met/Trp/stop excluded per Peden 1999 — the ATGGCA GC3=50% vs GC3s=0% contrast), and OverallGc=(GC1+GC2+GC3)/3. CAI here re-validates [[codon-adaptation-index]] with the *skip-zero-w* deviation (vs the 1e-6 clamp); bundled `EColiOptimalCodons` (Sharp&Li 1987 w) / `HumanOptimalCodons` (Kazusa RSCU) tables never trigger it. Frame-1/DNA-only; non-ACGT codons skipped. Validation [[codon-stats-001-report]].
- [[genetic-code-translation]] — the codon-family's foundational table: `GeneticCode` codon→amino-acid map (64 codons → 20 AA + 3 stops), 4 of NCBI's 33 tables (Standard/1, Vertebrate-Mito/2, Yeast-Mito/3, Bacterial-Plastid/11) as static singletons; `Translate`/`IsStartCodon`/`IsStopCodon`/`GetCodonsForAminoAcid`; DNA↔RNA T→U + case-insensitive + stop→`*` + len≠3→ArgumentException; Met/AUG & Trp/UGG single-codon fixed points (the w≡1 fact behind CAI/optimization); IUPAC ambiguity codon→`'X'` (a flagged divergence from the Evidence doc's promised exception). The table [[open-reading-frame-detection|ORF detection]] and [[codon-optimization]] read. Also covers the **whole-sequence `Translator` layer** (TRANS-PROT-001 + TRANS-SIXFRAME-001): framed / six-frame translation + genetic-code-parameterized START→STOP ORF finding, verified against the UniProt-P01308 insulin B-chain oracle; six-frame reverse-frame numbering follows the **Biopython independent-offset** convention (−k = RC offset k−1), the documented alternative to EMBOSS phase-locked.
- [[repetitive-element-detection]] — repeats/tandem family anchor: tandem (head-to-tail, ≥2 copies, primitive unit), inverted (W·G·W̄ᴿ / zero-gap palindrome, IUPACpal), and RepeatMasker-class assignment; ClassifyRepeat = exact-substring containment, not Smith-Waterman homology.
- [[consensus-sequence]] — assembly CONSENSUS family anchor: column-wise majority/threshold consensus (Biopython dumb_consensus rule) — gaps skipped, emit iff unique max and freq ≥ threshold else ambiguous, tie→ambiguous, all-gap→N no div-by-zero; N/0.5 parameterized defaults; shared with MSA consensus step.
- [[consensus-from-alignment]] — Motif-Analysis consensus-from-a-multiple-alignment (MOTIF-CONS-001, `MotifFinder.CreateConsensusFromAlignment`): pure most-frequent (plurality) column consensus over equal-length aligned strings, deterministic alphabetical tie-break (A<C<G<T), no threshold (always emits); Rosalind CONS `ATGCAACT` oracle; `alternative_to` the threshold/tie→ambiguous assembly [[consensus-sequence]], motif-family sibling of the exact [[known-motif-search]].
- [[iupac-degenerate-consensus]] — MotifFinder degenerate-consensus unit (MOTIF-GENERATE-001, `MotifFinder.GenerateConsensus`): collapses equal-length aligned columns into an IUPAC-ambiguity-coded string — per column keep every base with `count > 0.25·n` (strict `>`), emit the NC-IUB 1984 symbol for that base set (R/Y/S/W/K/M, B/D/H/V, N), no-pass fallback → most-frequent (alphabetical tie); DECIPHER threshold-consensus family + NC-IUB/UCSC/Wikipedia bijective set→symbol table; `A,A,G,G,C`→R / strict-`>` boundary / four-way→A-fallback oracles; `alternative_to` the plurality [[consensus-from-alignment]], degenerate sibling of the exact [[known-motif-search]] and de-novo [[overrepresented-kmer-discovery]].
- [[iupac-degenerate-matching]] — pattern-matching PAT-IUPAC-001 (`MotifFinder.FindDegenerateMotif` / `IupacHelper.MatchesIupac`): the **matching** direction of the IUPAC vocabulary — brute-force slide of an ambiguity-coded motif over a DNA subject, emitting each window `S[i..i+m)` whose every base falls in the allowed-set of the motif code at that position (15 standard codes A/C/G/T/N/R/Y/S/W/K/M/B/D/H/V), fixed `Score = 1.0` (match/non-match only, no weighting), 0-based `Position`, `O(n·m)` scan, cancellation checked every 1000 starts; guards = null `DnaSequence`→ArgumentNullException, empty/over-long motif→no matches, lowercase→uppercased, invalid code→ArgumentException/ArgumentOutOfRangeException (rejected, never literal); E-box `CANNTG` / Kozak `GCCGCCRCCATG` oracles; the `alternative_to` the exact [[known-motif-search]] (relaxes per-position equality to base-set membership), the consumer of the code produced by the generation-direction [[iupac-degenerate-consensus]], and the general engine behind the fixed-catalog E-box scan in [[regulatory-element-detection]].
- [[position-weight-matrix]] — pattern-matching PAT-PWM-001 (`MotifFinder.CreatePwm` / `ScanWithPwm`, `PositionWeightMatrix`): the **weighted-scoring** motif branch — build a log-odds PWM/PSSM from equal-length aligned DNA (PFM counts → PPM `(count+p)/(N+4p)` pseudocount-smoothed frequency, default `p=0.25` → `log₂(PPM/0.25)` against a **fixed uniform 0.25 background**), fixed `double[4,L]` A/C/G/T row layout; scan slides the matrix and emits each window scoring `>= threshold` (inclusive, default `0.0`) as a `MotifMatch` with 0-based `Position`, summed log-odds `Score`, and `Pattern = pwm.Consensus`; derived `Consensus` = per-column max base (INV-02), `MaxScore`/`MinScore` = Σ column extrema (INV-03), `Length` = training width (INV-01); guards = null→ArgumentNullException, empty/unequal-length/non-ACGT training→ArgumentException, subject shorter than PWM→no matches; simplifications = uniform-0.25 background only (no GC-bias), DNA-only 4-row (no protein), no gaps/profile-HMM, raw scores not p-values; `alternative_to` the single-best-base [[consensus-from-alignment]] (PWM keeps the full per-column weight vector; its own Consensus field is that same best-base collapse), weighted sibling of the exact [[known-motif-search]] and degenerate [[iupac-degenerate-matching]]; worked instance = the `AcceptorPwm` scanner in [[splice-acceptor-site-prediction]].
- [[suffix-tree]] — the **data structure** beneath the pattern-matching / repeat / comparison family (`SuffixTree`, `SuffixTree.Persistent`, `SuffixTree.Core`): a compressed trie of all suffixes of `T` built by **Ukkonen's online algorithm** (`O(n)` amortized — active point `(node, edge, length)` + remainder, three extension rules, **suffix links** for amortized `O(1)` jumps, `−1` terminator forcing explicit leaves), then `O(m)` substring queries. Post-build passes precompute **bottom-up `LeafCount`** (→ `O(1)` CountOccurrences) and the **deepest internal node** (→ `O(1)` LRS length). **Two backends, one `ISuffixTree`**: in-memory heap nodes (.NET 8, hybrid inline≤4→`Dictionary` children, SIMD `SequenceEqual` edges) and persistent **MMF v6** (.NET 9, 24-byte compact→32-byte large node transition, deepest-node/LRS depth stored in header bytes 72/80); shared algorithms are generic over a `struct`-constrained `ISuffixTreeNavigator<TNode>` for JIT specialization. Hosts the query units [[exact-pattern-search]] (Contains/Count/FindAll), [[longest-repeated-substring]] (deepest node ≥2 leaves), [[longest-common-substring]] (generalized tree), and the MEM/anchor engine `FindExactMatchAnchors` (MUMmer-style) feeding [[dot-plot-word-match]]; plus SHA-256 logical-hash + v2 stream serialization. Ukkonen 1995 / Gusfield 1997 / Delcher 1999 (MUMmer); no algorithm deviations (dual-backend + navigator engineering only). Primary spec `docs/algorithms/Pattern_Matching/Suffix_Tree.md`.
- [[regulatory-element-detection]] — Motif-Discovery regulatory-element unit (MOTIF-REGULATORY-001, `KnownMotifs` catalog scan): matches a **fixed, cited catalog of 12 canonical regulatory consensus elements** (TATA/−10/−35/CAAT/GC promoter boxes, Kozak + Shine-Dalgarno translation signals, poly(A), and E-box/AP-1/NF-κB/CREB TF sites) against a DNA sequence, reporting each occurrence's Name/Pattern/Sequence at its 0-based start; mixes exact and one IUPAC-degenerate (E-box `CANNTG`) match; anchors the AP-1 corrected-consensus regression (`TGAGTCA`→`TGACTCA`, `AATGAGTCAGG`→0 hits); two source-backed representative-site assumptions (NF-κB `GGGACTTTCC`, Kozak `GCCGCCACCATGG`); the curated-biological-catalog specialization of the generic exact [[known-motif-search]], distinct from the generation-direction [[iupac-degenerate-consensus]] and de-novo [[overrepresented-kmer-discovery]]; the scored −10/−35 alternative is [[promoter-detection]].
- [[promoter-detection]] — Annotation-layer scored −10/−35 promoter motif detector (ANNOT-PROM-001, `GenomeAnnotator.FindPromoterMotifs`): scans DNA for the prokaryotic −35 box `TTGACA` and −10/Pribnow box `TATAAT` plus derived prefix-5/suffix-5/prefix-4 variants, scoring each hit `Σ matched-position p / Σ all-6 consensus p ∈ [0,1]` from the E. coli position-specific frequencies (−35 sum 373, −10 sum 412); a full box scores 1.0, partial variants 0.665–0.855; `position` is a **0-based string index** (not the TSS-relative negative coordinate), `type` ∈ {`-35 box`,`-10 box`}, −35/−10 scanned independently (no pairing). Declared scope limits: **17 bp spacing not enforced**, **exact-substring not PWM/HMM/mismatch-tolerant** — motif annotation, not full promoter prediction; no null guard. The `alternative_to` the exact-hexamer catalog scan [[regulatory-element-detection]] (same two boxes, no scoring). Validation report [[annot-prom-001-report]].
- [[overrepresented-kmer-discovery]] — Motif-Discovery de-novo unit (MOTIF-DISCOVER-001, `MotifFinder.DiscoverMotifs`): finds **unknown** over-represented motifs by enumerating every length-`k` k-mer, counting overlapping occurrences, and ranking by observed/expected enrichment `Count/((N−k+1)/4^k)` under a zero-order i.i.d. uniform background (Compeau & Pevzner); deterministic exact single-pass, 0-based positions; `ATGC` in `ATGCATGCATGC` k=4 → 768/9≈85.333, `AAA` in `AAAAAAAAAA` k=3 → 64.0 oracles; `k>N`→empty, minCount presentation-only, zero-order-only + no-p-value simplifications; the de-novo inverse of the exact [[known-motif-search]] and distinct from the [[consensus-from-alignment]] plurality consensus.
- [[shared-motifs]] — Motif-Discovery cross-sequence unit (MOTIF-SHARED-001, `FindSharedMotifs`): the van Helden / RSAT oligo-analysis **"matching sequences"** quorum — enumerate every fixed-`k` exact word across a *set* of sequences and report each word present in ≥ `minSequences` of them (each carrying its `SequenceIndices` set + `Prevalence`=matching/total); a within-sequence repeat counts **1** (presence/absence, not multiplicity); exact-word only (no degenerate/substitution); the cross-sequence sibling of the single-sequence O/E [[overrepresented-kmer-discovery]] and `alternative_to` the fixed-k-quorum-vs-single-longest LCSM framing of [[longest-common-substring]] (returns all qualifying words, not one longest present in all).
- [[approximate-pattern-matching-mismatches]] — pattern-matching PAT-APPROX family anchor (PAT-APPROX-003): Hamming-distance **approximate matching** — a `k`-mer appears with ≤`d` mismatches when `HammingDistance(P,P')≤d` (BA1H); `Count_d`/`FindApproximateOccurrences` scan windows, `FindFrequentKmersWithMismatches` (BA1I) ranks `k`-mers by their **d-neighborhood** (Hamming-ball) tally so the winner need not occur exactly, `Neighbors` (BA1N) enumerates the ball (includes identity), `FindBestMatch` returns the leftmost min-Hamming window; oracles {GATG,ATGC,ATGT} count 5, {6,7,26,27,78}, Count_1=4, ACG/1→10; d=0 degenerates to exact so it is the mismatch-tolerant `alternative_to` [[k-mer-positions]] and the fuzzy sibling of the exact [[known-motif-search]] / de-novo [[overrepresented-kmer-discovery]]; substitution-only (no indels — that is the alignment family), deviations none.
- [[edit-distance]] — pattern-matching PAT-APPROX-002: **Levenshtein (edit) distance** — min insertions+deletions+substitutions (each cost 1) to transform one string into another; space-optimized two-row Wagner–Fischer DP (`EditDistance`, O(m·n)/O(n), case-sensitive) plus `FindWithEdits` brute-force variable-window approximate search (window band `p−e … p+e` absorbs indels, uppercased/case-insensitive, `MismatchType` = Substitution vs Edit); invariants d=0⇔identical, d≥‖len diff‖, d≤max(len), and d≤Hamming on equal-length; the **indel-tolerant `alternative_to`** the substitution-only [[approximate-pattern-matching-mismatches]] (same `ApproximateMatcher.cs`/`ApproximateMatchResult`), metric dual of unit-cost [[global-alignment-needleman-wunsch]]; simplifications = no traceback (indel `MismatchPositions` empty), brute-force scan, no Damerau transpositions; `DnaSequence` overload has no null guard; `kitten→sitting=3` oracle, deviations none.
- [[exact-pattern-search]] — pattern-matching PAT-EXACT-001 **engine layer**: the three core `SuffixTree` exact-search primitives + two DNA wrappers. `Contains(P)` (≥1 occurrence, O(m)), `CountOccurrences(P)` (reads precomputed node `LeafCount`, O(m), no leaf walk), `FindAllOccurrences(P)` (collects `z` leaves, O(m+z)) over an O(n) Ukkonen suffix tree where `P` occurs at `i` iff `P` prefixes the suffix at `i`; INV-01 every hit `T[i..i+m)=P` / INV-02 Count=FindAll.Count / INV-03 Contains⇔≥1. THE sharp edge = empty-pattern split: core tree treats `""` as "occurs everywhere" (`Contains`→true, `Count`→text length, `FindAll`→`[0..n-1]`) and `null`→ArgumentNullException, but both DNA wrappers guard null/empty motif→empty. Two wrappers differ only in ordering (non-correctness): `MotifFinder.FindExactMotif` **sorts** positions, `GenomicAnalyzer.FindMotif` returns the suffix-tree list **unsorted**; both uppercase (case-insensitive DNA). Impl: hybrid SIMD edge compares ≥8 chars, precomputed LeafCount, thread-static buffers. Overlaps all reported (`ana`/`banana`→{1,3}, `ATAT`/`GATATATGCATATACTT`→{1,3,9}); Gusfield/Ukkonen/Rosalind SUBS, deviations None. The exact-equality engine **beneath** the multi-motif [[known-motif-search]] and single-string [[longest-repeated-substring]], also backing [[longest-common-substring]]/[[dot-plot-word-match]]; approximate/degenerate siblings redirect to [[approximate-pattern-matching-mismatches]]/[[edit-distance]]/[[iupac-degenerate-consensus]].
- [[coverage-depth-calculation]] — assembly COVER family anchor: per-base sequencing depth over a reference (count of placed reads spanning each position, exact); average depth = Σdepth/G = Lander-Waterman C=LN/G, breadth = covered/G = 1−e^−c; boundary-clip + empty-input rules; ACGTTGCAAT→1.5/1.0 oracle; Poisson (e^−c gap) as a property check only; read placement (minOverlap best-match) out of scope.
- [[de-bruijn-graph-assembly]] — assembly DBG family anchor: k-mer graph (distinct (k-1)-mers = nodes, each k-mer = one prefix→suffix edge, directed multigraph) + Eulerian-walk reconstruction (balanced/semi-balanced, Euler theorems 8.1/8.2, O(|E|) Hierholzer); spelling = path[0]+last char of each node; unique-walk oracles only (AAABBBA, a_long…, to_every… k=4, ATGGCGTGCA); repeat/gap/error → non-Eulerian; walk-selection unspecified so exact asserts on unique-walk inputs; OLC is the alternative formulation.
- [[overlap-layout-consensus-assembly]] — assembly OLC family anchor: the three-stage Overlap-Layout-Consensus paradigm (`FindAllOverlaps` + `AssembleOLC`) — overlap graph (read=node, edge on longest suffix-of-A/prefix-of-B ≥ threshold), Layout = Hamiltonian path (NP-complete) + transitive reduction + non-branching contigs, Consensus = majority vote; GTACGTACGAT 12-edge / 5-overlap-tiling oracles; suffix-tree O(N+a) vs all-pairs O(N²); NP-complete/repeat-split/greedy-suboptimal failure modes; identity-1.0 + empty-input assumptions; Hamiltonian/overlap-graph alternative to DBG's Eulerian/k-mer formulation.
- [[contig-merge-overlap-collapse]] — assembly MERGE family anchor: suffix–prefix overlap collapse primitive (`MergeContigs`) behind greedy SCS and OLC layout; overlap = length-l suffix of X = prefix of Y, longest match reported, collapse keeps one copy so |merge|=|c1|+|c2|−l; overlap 0 / out-of-range → plain concatenation; BAA+AAB→BAAB, chain→AAABBBA oracles; caller-trusts-length + out-of-range→concat API-contract assumptions; overlap-based sibling of DBG's k-mer/Eulerian formulation.
- [[scaffolding]] — assembly SCAFFOLD family anchor: joining ordered contigs into a scaffold, adjacent contigs separated by `N`-runs of length = the (upstream) inter-contig distance estimate (ABySS 2.0 rule); positive gap `g` → `g`×`N`, scaffold length = Σ|contig|+Σgap; non-positive (zero/negative) estimate → AGP unknown-size default 100 N (negative = overlap, unresolved here); each contig in ≤1 scaffold, unlinked → singleton; parameterized fill char; ACGTNNNTTGGNNCCAA / AAAA+100N+TTTT oracles; downstream of DBG/OLC/merge, hands off to contig-merge on the negative-gap overlap case; 100-constant source-backed (INSDC), only the fall-back-not-resolve decision is assumed.

- [[ancestry-estimation-admixture]] — anchor for the **population-genetics** `POP-*` family (**first POP-\* unit**, POP-ANCESTRY-001): estimate an individual's **ancestry proportions Q** (genome fraction from each of K reference populations) from unlinked SNP genotypes via **supervised/projection ADMIXTURE** — reference allele frequencies **F fixed/known**, only Q estimated by EM. Alexander, Novembre & Lange 2009 model: genotype g∈{0,1,2}=allele-1 copy count, **log-likelihood Eq. 2** `L=ΣᵢΣⱼ{g·ln(Σ_k q·f)+(2−g)·ln(Σ_k q·(1−f))}` under simplex `Σ_k q_ik=1`; **FRAPPE EM Eq. 4** `q^{n+1}=(1/2J)Σⱼ[g·a_ijk+(2−g)·b_ijk]` (a/b = posteriors an allele-1/allele-2 copy came from k), EM monotone ascent, convergence Eq. 5 ΔL<ε=10⁻⁴, O(IJK²); oracles symmetric K=2/J=2 f_A=[0.8,0.2]/f_B=[0.2,0.8] g=[2,0] → one EM iter **(0.8,0.2) exactly** → converges (1.0,0.0), single-SNP g=2→(0.9,0.1)/g=1→(0.5,0.5) uninformative-het/g=0→(0.1,0.9), identical panels uniform fixed point; INV simplex Σ_k q=1 & EM L non-decreasing, missing g∉{0,1,2} skipped, empty inputs→empty; **label non-identifiability** (K! maxima under population relabeling) **pinned by fixed labelled reference panels** (ADMIXTURE manual §2.10/§2.14) so Q interpretable relative to supplied labels; scope = estimate-Q-given-fixed-F only (does NOT jointly learn F, run unsupervised block-relaxation, choose K, or model linkage); **distinct from** the tree-building phylogenetics `PHYLO-*` family ([[distance-based-tree-construction]], [[evolutionary-distance-matrix]]) — a mixture-weight/EM decomposition of one admixed individual, not a tree; two research-grade assumptions (maxIterations budget + ε convergence; skip-missing); no contradictions; record [[pop-ancestry-001-evidence]].
- [[genetic-diversity-statistics]] — population-genetics `POP-*` family diversity-statistics unit (**second POP-\* unit**, POP-DIV-001, sibling of the anchor [[ancestry-estimation-admixture]]): four classical within-population variation scalars from an aligned sample (n seqs, length L, S segregating sites) — **nucleotide diversity π** = Σ_{i<j} d_ij/(C(n,2)·L) (Nei & Li 1979, mean per-site pairwise differences), **Watterson's θ_W** = S/a_n with harmonic a_n = Σ_{i=1}^{n−1} 1/i (per-site S/(a_n·L); Watterson 1975), **Tajima's D** = (k̂ − S/a_1)/√(e_1·S+e_2·S(S−1)) (Tajima 1989 — normalized π-vs-Watterson difference; **k̂ = pairwise-difference COUNT not per-site π**; D≈0 neutral, D<0 excess rare alleles→sweep/expansion, D>0 deficit→balancing/contraction), and **heterozygosity/gene diversity** H_exp = (1/L)Σ(1−Σp²) with unbiased haploid H_obs = n/(n−1)·H_exp ≡ π (Nei 1978); Wikipedia oracle n=5/L=20/S=4 → k̂=2.0/π=0.1/S/a_1≈1.92/θ_W≈0.096/**D≈0.273** (TD-C01/TD-C02); guards n<3→D undefined→0, S=0/monomorphic→all 0, Var≤0→D=0, `CalculateTajimasD(k̂,S,n)` expects k̂ not π; **distinct from** the anchor's ancestry-proportion EM (within-population *variation amount* vs *ancestry decomposition*), summary-only (no variant calling / phasing / recombination / Fu-Li / Fay-Wu); all formulae exact-match, no deviations; record [[pop-div-001-evidence]].
- [[allele-genotype-frequencies]] — **foundational** population-genetics `POP-*` primitive (POP-FREQ-001): turn per-locus **genotype counts** into **allele frequencies**, derive **minor allele frequency (MAF)**, and **filter variants by MAF** — the numeric substrate under the rest of the POP family (the fixed reference frequencies **F** of [[ancestry-estimation-admixture]] and the per-site `p_i` behind the heterozygosity term of [[genetic-diversity-statistics]] are exactly these quantities). Biallelic allele freq `p = f(AA)+½f(AB)` / `q = f(BB)+½f(AB)` via counts `total=2·(n_AA+n_AB+n_BB)`, `major=2·n_AA+n_AB`, `minor=2·n_BB+n_AB` (**INV `p+q=1`**, `major+minor=total`; oracle 49/42/9 → 0.70/0.30); VCF/PLINK dosage `alt_freq = Σg/(2n)`, `MAF = min(alt_freq, 1−alt_freq)` (**INV `0 ≤ MAF ≤ 0.5`**, symmetric — alt_freq 0.7 → MAF 0.3; monomorphic → 0; 50/50 → 0.5); MAF filter = inclusive `[minMAF,maxMAF]` band with conventional HapMap/common(>0.05)-vs-rare(<0.05) thresholds; edge cases zero samples→(0,0), empty vector→MAF 0, **negative count → `ArgumentOutOfRangeException`**; scope = biallelic counting/normalization only (no HWE test, no multiallelic, no phasing/imputation); no deviations/contradictions; record [[pop-freq-001-evidence]].
- [[population-differentiation-fst]] — population-genetics `POP-*` differentiation unit (POP-FST-001): measure **how differentiated populations are** — fraction of total variation due to between-population structure — from per-population allele frequencies (**consumes** [[allele-genotype-frequencies]]; distinct from within-sample [[genetic-diversity-statistics]] and from ancestry decomposition [[ancestry-estimation-admixture]]). **Wright 1965 variance Fst** `Fst = σ_S²/(pBar(1−pBar))`, two-pop size-weighted `pBar=(n1·p1+n2·p2)/(n1+n2)` / `σ_S²=(n1(p1−pBar)²+n2(p2−pBar)²)/(n1+n2)`, multi-locus **ratio-of-sums** `Σσ_S²/Σhet` — the **population parameter from known allele freqs**, explicitly **NOT** the Weir & Cockerham 1984 θ estimator (no bias correction); **F-statistics** heterozygosity partition `Fis=1−Hi/Hs`/`Fit=1−Hi/Ht`/`Fst=1−Hs/Ht` with **exact** `(1−Fit)=(1−Fis)(1−Fst)`; INV Fst∈[0,1] (0=panmixia/1=fixed differences), Fis∈[−1,1] (negative under excess heterozygotes), pairwise matrix symmetric + zero diagonal, **not a metric** (fails triangle inequality); oracles fixed p1=1/p2=0→**1.0 exactly**, pop1=(.9,.8)/pop2=(.1,.2)→1/2, Cavalli-Sforza/Elhaik ref values, Hartl-Clark bands (0.05/0.15/0.25); edge cases denominator 0 (empty/both-fixed-same/monomorphic)→**return 0**; scope = Wright Fst + heterozygosity F-statistics only (no significance test, no bootstrap CI); no deviations/contradictions; record [[pop-fst-001-evidence]].
- [[hardy-weinberg-equilibrium-test]] — population-genetics `POP-*` unit (POP-HW-001): the **Hardy-Weinberg equilibrium chi-square test** — does a biallelic locus's observed genotype counts match the `p²/2pq/q²` proportions expected under random mating? **Consumes** genotype counts from [[allele-genotype-frequencies]] (POP-FREQ-001, whose scope explicitly leaves the HWE test here) and turns them into a **hypothesis test**. Estimate `p = (2·n_AA + n_Aa)/(2n)` / `q = 1−p`, expected counts `E = {p²n, 2pqn, q²n}`, **Pearson χ² = Σ (O−E)²/E** over the 3 genotype classes, **df = 1** (`#genotypes − #alleles = 3 − 2`), p-value from chi-square CDF (lower-incomplete-gamma approx), default **α = 0.05** critical value **3.841**; returns `InEquilibrium`/`ChiSquare`/`PValue`. Oracles Ford moth (1469,138,5)→`p≈0.954`/`χ²≈0.83`→in-eq, perfect (25,50,25)→`χ²=0`, excess-het (10,80,10)→`χ²=36 ≫ 3.84`→out-of-eq; edge cases **zero samples→InEquilibrium true, PValue 1** (no evidence against H₀), monomorphic→`χ²=0`, **expected-0 term skipped** (div-by-zero guard); deviation causes non-random mating/selection/mutation/migration/drift/**genotyping error**; scope = biallelic chi-square only (no Wigginton 2005 **exact test**, no multiallelic); no deviations/contradictions; record [[pop-hw-001-evidence]].
- [[linkage-disequilibrium]] — population-genetics `POP-*` unit (POP-LD-001): **non-random association between alleles at two loci** — a **pairwise inter-locus** quantity distinct from per-locus counting ([[allele-genotype-frequencies]]), within-sample diversity ([[genetic-diversity-statistics]]), between-population differentiation ([[population-differentiation-fst]]), and the single-locus [[hardy-weinberg-equilibrium-test]]; **consumes** allele/haplotype frequencies from [[allele-genotype-frequencies]]. `D = p_AB − p_A·p_B` (Robbins 1918), normalized `D' = |D|/D_max` clamped [0,1] (Lewontin 1964, sign-branched `D_max`) and `r² = D²/(p_A·q_A·p_B·q_B)` (Hill & Robertson 1968); implemented **phase-free** via the diploid-frequency result `R_AB=r_AB` (Wright 1933) — `r²` = squared Pearson correlation of 0/1/2 genotype dosage vectors `Cov²/(Var·Var)`, `D` recovered from `Cov_diploid=2D ⇒ D=Cov/2`; `FindHaplotypeBlocks` = simplified adjacent-pair Gabriel 2002 (consecutive `r² ≥ threshold`, default **0.7**, ≥2 variants); INV `0≤r²≤1`/`0≤|D'|≤1`, empty→`r²=0,D'=0`, monomorphic/zero-variance→`r²=0` (guarded div), distance+IDs preserved, blocks Start≤End/≥2/non-overlapping/position-ordered; perfect correlation **and** perfect anti-correlation both →`r²=1`/`D'=1` (sign-blind); scope = two-biallelic-loci r²/D' + adjacent-pair blocks only (no full LD matrix, no phasing/EM, no LD-decay fit, no exact Gabriel CI); no deviations/contradictions; record [[pop-ld-001-evidence]].
- [[runs-of-homozygosity-inbreeding]] — population-genetics `POP-*` unit (POP-ROH-001): detect **long uninterrupted homozygous stretches** (**runs of homozygosity**, `FindROH`) in one individual and summarize their burden as the genomic inbreeding coefficient **F_ROH** — a **per-individual segment-detection** problem distinct from per-locus counting ([[allele-genotype-frequencies]]), within-sample diversity ([[genetic-diversity-statistics]]), between-population differentiation ([[population-differentiation-fst]]), two-locus [[linkage-disequilibrium]], and single-locus [[hardy-weinberg-equilibrium-test]]. **Window-free consecutive-runs** scan (Marras 2015 / detectRUNS): walk position-sorted 0/1/2 genotypes (1 = heterozygous = "opposite"), grow a run, **terminate** when opposite calls exceed `maxOppRun` (a few tolerated to absorb genotyping error) or an inter-SNP gap exceeds `maxGap` (breaks even an all-homozygous run); retain a stretch only if it passes **both** `minSNP` **and** `minLengthBps` (PLINK `--homozyg-snp 100` **and** `--homozyg-kb 1000`, one alone insufficient). **F_ROH = ΣL_roh / L_auto** (McQuillan 2008 verbatim; `L_auto` = SNP-covered autosome ≈2,674 Mb; r=0.86 vs pedigree; min-length lever ≥0.5/1.5/5 Mb, 1.5 Mb best); oracle ΣL_roh=20 Mb/L_auto=100 Mb → **0.20**, whole-genome → 1.0; INV each run once/Start≤End/SnpCount, one tolerated het keeps one run / het beyond splits, gap>maxGap breaks, below minSNP-or-minLength discarded, unsorted ordered internally, genotype 2 = homozygous, leading hets skipped, F_ROH∈[0,1]; two API-encoding assumptions (0/1/2 encoding; **missing-genotype handling out of scope** — no missing sentinel, non-`1` treated homozygous); scope = consecutive-runs scan + F_ROH only (no PLINK sliding-window scoring, no missing bounding, no LOD/model-based callers, no cohort ROH-island mapping); no deviations/contradictions; record [[pop-roh-001-evidence]].
- [[selection-scan-ihs-ehh]] — population-genetics `POP-*` unit (POP-SELECT-001): detect **recent positive selection** from **haplotype structure** — a selective sweep leaves an unusually long, low-diversity haplotype around the selected allele. - [[primer-design]] — PCR primer-design `PRIMER-*` / MolTools family **orchestrator** (PRIMER-DESIGN-001, `DesignPrimers`): the **end-to-end primer-PAIR design pipeline** — a fast heuristic selector, **not** a Primer3 combinatorial pair optimizer (`Implementation Status: Simplified`). Four stages: (1) **search regions** ≤ 200 bp upstream of `targetStart` / ≤ 200 bp downstream of `targetEnd`; (2) **candidate generation** over the length range (default 18–25 bp), reverse-region candidates **reverse-complemented before scoring** (primer orientation); (3) **greedy per-side selection** — best forward & best reverse chosen **independently** by the per-primer `EvaluatePrimer`/`CalculatePrimerScore` heuristic (**higher-is-better**, base 100: `100 − 2|len−opt| − 2|Tm−opt| − 0.5|GC%−50| − 5·homopolymerLen + 5 GC-clamp bonus when final base G/C`); (4) **pair compatibility** — accept iff `|Tm_f − Tm_r| ≤ 5 °C` **AND** `!HasPrimerDimer` (INV-02). `ProductSize = reverse.Position + reverse.Length − forward.Position` (INV-03). This legacy additive `Score` — **not** the validated sign-gated *lower-is-better* Primer3 objective [[primer3-weighted-penalty-objective]] (`CalculatePrimer3Penalty`) — is what actually ranks the pair; PRIMER-DESIGN-001 *is* the "separate (future) pair-level concern" that page defers. **Consumes** the PRIMER-TM-001 Tm + primer-dimer signals of [[primer-dimer-thermodynamics-tm]] (PRIMER-STRUCT-001) as its compatibility gate; primer analogue of the single-oligo [[hybridization-probe-design]]. Defaults `DefaultParameters`: 18–25 bp/`OptimalLength=20`, 40–60 % GC, **57–63 °C**/`OptimalTm=60`, `MaxHomopolymer=4`, `MaxDinucleotideRepeats=4`, `Avoid3PrimeGC=false` (when on, flags final **two** bases with no G/C), `Check3PrimeStability=true` (emits `ΔG < −9` issue). Classic count-normalization, not kallisto/salmon effective-length or edgeR/DESeq2 size factors. Research-grade, not for clinical use. Record [[trans-expr-001-evidence]].
- [[alternative-splicing-psi]] — **Transcriptome / RNA-seq** family, **alternative / differential splicing** unit (TRANS-SPLICE-001): quantify how a gene's exons are spliced from RNA-seq reads with `TranscriptomeAnalyzer.CalculatePSI` / `DetectAlternativeSplicing`. **Percent-Spliced-In** `Ψ = I/(I+S)` (inclusion vs exclusion read support; Wang 2008, PMC3330053, SUPPA2) with opt-in **rMATS length-normalized** `ψ̂ = (I/lᵢ)/(I/lᵢ + S/lₛ)` (Shen 2014); **ΔPSI** = splicing-level differential readout. Classifies an event between **two isoforms of one gene** into the **five canonical AS classes** (Wang 2008 = rMATS SE/RI/A5SS/A3SS/MXE): SkippedExon, RetainedIntron, AlternativeFivePrimeSS (alternative **donor**), AlternativeThreePrimeSS (alternative **acceptor**), MutuallyExclusiveExons. Oracles: (80,20)→0.80, (80,20,200,100)→0.6666…. Edge cases: 0/0→NaN, S=0→1, I=0→0, `0≤PSI≤1`, <2 isoforms → no event. The **splicing-level counterpart** to the gene-level anchor [[differential-expression]] (a gene can be differentially spliced without changing total expression); self-normalizing ratio, so unlike [[expression-quantification]] it needs no TPM/library correction. **Distinct from** the genomic splice-site *motif* predictors [[splice-donor-site-prediction]] / [[splice-acceptor-site-prediction]] / [[gene-structure-prediction-intron-exon]] (which score `GT`/`AG` sequence signals) — cross-linked only to disambiguate. Two source-backed ASSUMPTIONs (length normalization opt-in; forward strand). Read-count ratio, **not** the full rMATS/SUPPA2 replicate statistics. Research-grade, not for clinical use. Record [[trans-splice-001-evidence]].
- [[gene-structure-prediction-intron-exon]] — **splicing** family **composite / umbrella** (SPLICE-PREDICT-001): **gene structure prediction (intron/exon)** — the integrative unit that **consumes** [[splice-donor-site-prediction]] + [[splice-acceptor-site-prediction]] and adds what neither boundary unit produces: **pairs** each `GT/GU` donor with an `AG` acceptor into an **intron** under the GT-AG rule (bounded by `minIntronLength` default 60 / `maxIntronLength`, **greedy non-overlapping** selection), then builds the whole model — **exons** = the gaps, each **typed** Initial/Internal/Terminal/Single and carrying a reading-frame **phase** = `(Σ preceding exon lengths) mod 3`, the **spliced (mature) sequence** (introns removed), and an **overall score** = mean of per-intron `(donor.Score+acceptor.Score)/2` in [0,1]. U2 (GT-AG) / GC-AG / U12 (AT-AC) intron types; single-exon (intronless) genes → 1 Single exon; empty → empty structure. Sources Breathnach & Chambon 1981 + Shapiro & Senapathy 1987 + Burge 1999 + Wikipedia Intron/Exon/Gene-structure; all 4 assumptions RESOLVED (default branch-point 0.3 constant removed). Record [[splice-predict-001-evidence]].
- [[variant-effect-annotation-vep]] — the **interpretation / annotation** member of the variant-analysis family (VARIANT-ANNOT-001): **VEP-style variant effect annotation** — map an already-**called** variant to its **functional consequence** + **Sequence-Ontology term/accession** + **IMPACT** (HIGH/MODERATE/LOW/MODIFIER) via the Ensembl **OverlapConsequence predicate system**. **IMPACT is a stored property of the term, not computed**; **most-severe selection returns the lowest-rank term**. The coding engine **`depends_on`** [[genetic-code-translation]] — **translates the ref/alt codons** (NCBI Standard code table 1) and compares peptides: `synonymous` / `missense` / `stop_gained` / `stop_lost` from `ref_pep` vs `alt_pep`, **`frameshift` purely length-based `|alt−ref| mod 3 ≠ 0`**, inframe ins/del = ×3 indel; precedence stop_gained>missense, start_lost(rank 7)>coding-substitution. Downstream of variant calling [[somatic-variant-calling-tumor-normal]], orthogonal to pathogenicity/tier classification. Sources McLaren 2016 (VEP) + Ensembl Constants.pm/VariationEffect.pm + NCBI gc.prt + Sequence Ontology. Two assumptions (Standard code table 1 only; single-codon SNV comparison, no NMD). Research-grade, not for clinical use. Record [[variant-annot-001-evidence]].
- [[germline-variant-calling-snp-indel]] — the **calling / detection** member of the variant-analysis family (VARIANT-CALL-001): **variant calling** — detect **SNP / insertion / deletion** (Danecek 2011: a variant is a difference from reference) by aligning a query to a reference (`CallVariantsFromAlignment` → `SequenceAligner.GlobalAlign`) and reading out differing columns, then classify each SNP **Transition** (A↔G, C↔T) vs **Transversion** (A↔C/A↔T/G↔C/G↔T) and report **Ti/Tv** (undefined `#Tv=0`→0). In-memory `Variant` uses a `"-"` gap sentinel + 0-based position; VCF **padding base + 1-based POS** apply only to serialized `ToVcfLines`. **Upstream of** annotation [[variant-effect-annotation-vep]] (calling produces what VEP interprets); **`alternative_to`** the oncology tumor-vs-normal caller [[somatic-variant-calling-tumor-normal]] (same goal, different evidence). Assumptions: indels **not** left-aligned/parsimony-normalized (Tan 2015 — position-only implementation-dependence in repeats), gap-sentinel internal rep, Ti/Tv zero→0. Alignment-based single-query comparison, **not** a read-pileup genotype caller. Sources VCFv4.3 + Danecek 2011 + Tan 2015 + Collins & Jukes 1994 + Acharya 2017 (Hamming). Research-grade, not for clinical use. Records [[variant-call-001-evidence]] + [[variant-snp-001-evidence]] (SNP facet: `FindSnps`/`FindSnpsDirect`, Hamming-mismatch enumeration, equal-length/common-prefix precondition) + [[variant-indel-001-evidence]] (indel facet: `FindInsertions`/`FindDeletions`, directional length invariant, Tan 2015 normalization theory).
