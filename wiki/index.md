## Concepts

- [[three-front-doors]] — one algorithm engine exposed through skills, the C# API, and MCP, with identical results.
- [[skill-layer]] — the Agent-Skill routing + discipline layer that keeps 427 tool schemas out of the model's context.
- [[layered-architecture]] — the strict up-only dependency layering (Levels 0–4) enforced by architecture tests.
- [[scientific-rigor]] — runtime honesty: `LimitationPolicy`, tool-only computation, provenance on every result.
- [[validation-and-testing]] — 22k+ tests across ten methodologies plus the per-unit validation campaign.
- [[build-quality-gate]] — the build-time static-analysis gate: SonarAnalyzer ratcheted to blocking under `TreatWarningsAsErrors`, fix-or-silence, review-not-blind-fix.
- [[test-unit-registry]] — the area-prefixed Test Unit ID scheme and per-unit record behind the validation effort.
- [[definition-of-done]] — the six-criterion acceptance bar (TestSpec, tests, ≥80% coverage, edge cases, CI, evidence) each unit must clear.
- [[algorithm-validation-evidence]] — the templated per-unit `docs/Evidence/` artifact pattern (sources, worked-example dataset, deviations) behind the "Evidence documented" DoD criterion.
- [[global-alignment-needleman-wunsch]] — the canonical DP global-alignment algorithm (`GlobalAlign`): linear-gap recurrence, O(nm), traceback, affine-as-extension.
- [[multiple-sequence-alignment]] — aligning 3+ sequences (NP-complete): Seqeron's star (`MultipleAlign`), iterative-refinement (MUSCLE), and consistency (T-Coffee) aligners; sum-of-pairs + consensus.
- [[semi-global-alignment-fitting]] — ends-free "glocal" alignment: fitting/query-in-reference variant (Rosalind SIMS); NW recurrence with free reference end gaps, traceback from max of last row.
- [[alignment-statistics]] — post-alignment metrics: percent identity/similarity/gaps (EMBOSS/BLAST convention, denominator includes gaps, positive-score ⇒ similar) and srspair three-line markup.
- [[coding-potential-hexamer-score]] — CPAT hexamer usage-bias coding-potential score: mean of ln(coding/noncoding) over in-frame hexamers (wordSize 6/step 3), sign convention, ±1 pseudo-scores; Fickett TESTCODE as not-implemented alternative.
- [[relative-synonymous-codon-usage]] — RSCU (Sharp & Li 1986) codon-usage-bias measure: n_i·x/Σx normalization, 1.0=no bias, bounded [0,n_i], Σ-over-family=n_i invariant; pooled counts, sense codons only; base anchor of the codon-usage family (vs CAI's max-normalization).
- [[codon-adaptation-index]] — CAI (Sharp & Li 1987) whole-gene codon-bias / expression-proxy score in [0,1]: geometric mean of relative adaptiveness w=f/max_synonym_f, low-value-sensitive, stop excluded; dual include/exclude-Met/Trp mode (exclude can give L=0→0); E. coli oracles; Seqeron deviation = 1e-6 zero-freq clamp (vs Sharp & Li's 0.5 reference-table pseudocount); one level above RSCU.
- [[repetitive-element-detection]] — repeats/tandem family anchor: tandem (head-to-tail, ≥2 copies, primitive unit), inverted (W·G·W̄ᴿ / zero-gap palindrome, IUPACpal), and RepeatMasker-class assignment; ClassifyRepeat = exact-substring containment, not Smith-Waterman homology.
- [[consensus-sequence]] — assembly CONSENSUS family anchor: column-wise majority/threshold consensus (Biopython dumb_consensus rule) — gaps skipped, emit iff unique max and freq ≥ threshold else ambiguous, tie→ambiguous, all-gap→N no div-by-zero; N/0.5 parameterized defaults; shared with MSA consensus step.
- [[coverage-depth-calculation]] — assembly COVER family anchor: per-base sequencing depth over a reference (count of placed reads spanning each position, exact); average depth = Σdepth/G = Lander-Waterman C=LN/G, breadth = covered/G = 1−e^−c; boundary-clip + empty-input rules; ACGTTGCAAT→1.5/1.0 oracle; Poisson (e^−c gap) as a property check only; read placement (minOverlap best-match) out of scope.
- [[de-bruijn-graph-assembly]] — assembly DBG family anchor: k-mer graph (distinct (k-1)-mers = nodes, each k-mer = one prefix→suffix edge, directed multigraph) + Eulerian-walk reconstruction (balanced/semi-balanced, Euler theorems 8.1/8.2, O(|E|) Hierholzer); spelling = path[0]+last char of each node; unique-walk oracles only (AAABBBA, a_long…, to_every… k=4, ATGGCGTGCA); repeat/gap/error → non-Eulerian; walk-selection unspecified so exact asserts on unique-walk inputs; OLC is the alternative formulation.
- [[overlap-layout-consensus-assembly]] — assembly OLC family anchor: the three-stage Overlap-Layout-Consensus paradigm (`FindAllOverlaps` + `AssembleOLC`) — overlap graph (read=node, edge on longest suffix-of-A/prefix-of-B ≥ threshold), Layout = Hamiltonian path (NP-complete) + transitive reduction + non-branching contigs, Consensus = majority vote; GTACGTACGAT 12-edge / 5-overlap-tiling oracles; suffix-tree O(N+a) vs all-pairs O(N²); NP-complete/repeat-split/greedy-suboptimal failure modes; identity-1.0 + empty-input assumptions; Hamiltonian/overlap-graph alternative to DBG's Eulerian/k-mer formulation.
- [[contig-merge-overlap-collapse]] — assembly MERGE family anchor: suffix–prefix overlap collapse primitive (`MergeContigs`) behind greedy SCS and OLC layout; overlap = length-l suffix of X = prefix of Y, longest match reported, collapse keeps one copy so |merge|=|c1|+|c2|−l; overlap 0 / out-of-range → plain concatenation; BAA+AAB→BAAB, chain→AAABBBA oracles; caller-trusts-length + out-of-range→concat API-contract assumptions; overlap-based sibling of DBG's k-mer/Eulerian formulation.
- [[scaffolding]] — assembly SCAFFOLD family anchor: joining ordered contigs into a scaffold, adjacent contigs separated by `N`-runs of length = the (upstream) inter-contig distance estimate (ABySS 2.0 rule); positive gap `g` → `g`×`N`, scaffold length = Σ|contig|+Σgap; non-positive (zero/negative) estimate → AGP unknown-size default 100 N (negative = overlap, unresolved here); each contig in ≤1 scaffold, unlinked → singleton; parameterized fill char; ACGTNNNTTGGNNCCAA / AAAA+100N+TTTT oracles; downstream of DBG/OLC/merge, hands off to contig-merge on the negative-gap overlap case; 100-constant source-backed (INSDC), only the fall-back-not-resolve decision is assumed.
- [[assembly-statistics]] — assembly STATS family anchor: contiguity/QC summary metrics over contig lengths (read-only, downstream of DBG/OLC/merge/scaffold) — Nx = shortest length covering ≥x% (largest-first, inclusive ≥; N50/N90), Lx = the matching contig count, N90 ≤ N50 / L90 ≥ L50, auN = ΣLᵢ²/ΣLⱼ (area under Nx curve, more stable than N50); + totals/largest/smallest/GC and 0-based inclusive N-run gaps; Miller 2010 + Wikipedia + QUAST + Heng Li 2020; Assembly A N50 70/L50 2, auN 73.333 oracles; empty→zeros (not None) + upper-median-not-source-derived assumptions.
- [[quality-trimming-running-sum]] — assembly TRIM family anchor: BWA/cutadapt running-sum quality trimming — subtract cutoff from each Phred quality, take partial sums to the 3' end, cut at the minimum (= BWA argmax of accumulated threshold−q, with `s<0` early break + `BWA_MIN_RDLEN=35` floor); Phred+33 decode (Cock 2010); `KI;<)(,%#$`@thr 10→first 4 bases oracle; threshold<1 disables / all-high unchanged / all-low dropped / good-base-among-bad refinement; both-end pass + downstream minLength filter as assumptions; read-QC preprocessing upstream of correction/DBG/OLC.
- [[kmer-spectrum-error-correction]] — assembly CORRECT family anchor: two-sided k-mer-spectrum read error correction (Musket/Quake) — trusted (multiplicity > cut-off) vs untrusted k-mers, base trusted if covered by any trusted k-mer, unique alternative making all covering k-mers trusted, ambiguity/no-set→unchanged; substitution-only (count+length preserved); kmerSize=15/minKmerFrequency=2 non-behavioral defaults.
- [[centromere-analysis]] — chromosome centromere/alpha-satellite family anchor (second Chromosome-analysis unit): `AnalyzeCentromere` + 4 additive detectors — Levan 1964 q/p arm-ratio classes (Metacentric ≤1.7 / Submetacentric ≤3.0 / Subtelocentric <7.0 / Acrocentric ≥7.0 / Telocentric p=0); alpha-satellite-specific `DetectAlphaSatellite`/`FindCenpBBoxes` (171-bp tandem period ±5, AT>0.5, 17-bp CENP-B box `YTTCGTTGGAARCGGGA`, no embedded monomer); `DetectHigherOrderRepeat` (HOR period = smallest k with monomers k apart ≥95% identical across ≥90%, intra-HOR 50–70% vs inter-HOR 97–100%); `AssignSuprachromosomalFamily` (bundled CC0 Dfam ALR/ALRa=A/ALRb=B, SF3 pentameric/SF4 monomeric/{SF1,SF2} dimeric/SF5 irregular); base AlphaSatelliteContent is density-heuristic-not-specific + SF1/SF2-unresolved (non-redistributable consensus library) limitations.
- [[karyotype-analysis]] — chromosome karyotyping/ploidy-detection family anchor (third Chromosome-analysis unit): `AnalyzeKaryotype` builds a karyotype from chromosome descriptors (split sex/autosome, group by base name, count copies, compare to expected ploidy, label nullisomy..pentasomy by absolute count; TotalChromosomes/TotalGenomeSize/MeanChromosomeLength invariants) and `DetectPloidy` estimates whole-genome ploidy from read depth (true-median ratio → round(ratio×2) clamped [1,8], confidence 1−|ratio×2−ploidy|×2, empty→(2,0)); Down/Turner + diploid/tetraploid/haploid oracles; DD4 nullisomy-unreachable-via-GroupBy, DD5 disomy-only-aneuploid-in-polyploid; deviations None. Descriptor-count sibling of depth-log-ratio [[aneuploidy-detection]].
- [[synteny-and-rearrangement-detection]] — shared synteny anchor (fourth Chromosome-analysis unit; comparative-genomics COMPGEN-SYNTENY will reuse it): `FindSyntenyBlocks` builds syntenic blocks from ortholog pairs (group by chr pair, sort by ref position, collinear runs merged under maxGap, emit ≥minGenes; block carries strand '+'/'-', GeneCount, SequenceIdentity=NaN for coordinate-only input per MCScanX; I1–I5) + `DetectRearrangements` classifies adjacent-block signals (diff target chr→Translocation, same chr diff strand→Inversion, gap asymmetry→Deletion, overlapping source diff targets→Duplication); forward/inverted-block + translocation(50000)/inversion(size 10000) oracles; empty/below-minGenes/maxGap-split/multi-pair edge cases; MCScanX/SyRI/MUMmer + Wikipedia; deviations None.
- [[telomere-analysis]] — chromosome telomere family anchor (fifth Chromosome-analysis unit): telomere repeat detection + length estimation — 3′ end forward TTAGGG / 5′ end reverse-complement CCCTAA (6-bp vertebrate unit, Meyne 1989), configurable 70%-per-window purity so imperfect repeats accepted (5/6 for 6-bp, 5/7 for Arabidopsis TTTAGGG; divergent TTAGGA→0.833), purity∈[0,1] tracked with length; length direct from run or qPCR T/S ratio (Cawthon 2002) EstimatedLength=referenceLength×(tsRatio/referenceRatio); IsCriticallyShort=(hasTelomere&&length<criticalLength)OR empty; criticalLength 3000 / minTelomereLength 500 / searchLength(truncates) / referenceLength 7000 configurable defaults not biological constants; Wikipedia/Meyne/Cawthon/Blackburn-Gall sources; deviations None. Sibling of [[aneuploidy-detection]].
- [[aneuploidy-detection]] — chromosome copy-number/ploidy family anchor (first Chromosome-analysis unit): abnormal chromosome count from read depth — per-bin CN=round(2^log2(depth/median)×2) clamped [0,10], whole-chromosome call needs dominant CN across ≥80% of bins (mosaicism tolerance), confidence=1−min(1,|CN/2−2^logRatio|)=1.0 at integer-CN ratios; nullisomy..pentasomy terminology; Down/Edwards/Patau/Turner/Klinefelter oracles; sex-chromosomes-not-special-cased + partial-aneuploidy-needs-80%-bins limitations.
