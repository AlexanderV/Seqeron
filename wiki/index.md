# Wiki Index

The catalog of all pages in this wiki. Each entry: a wikilink to the page and a one-line summary. The LLM reads this first when answering queries to identify candidate pages.

Keep summaries tight вҖ” one line each. The index is engineered to be cheap to read; a fat index defeats its purpose.

When this file exceeds ~300 lines or the wiki passes ~150 pages, shard into `wiki/indexes/<type>.md` and replace this file with a directory of shards. See the `scaling-playbook.md` reference in the `llm-wiki` skill for the migration procedure.

- [[validation-and-testing]] вҖ” 22k+ tests across ten methodologies plus the per-unit validation campaign.
- [[build-quality-gate]] вҖ” the build-time static-analysis gate: SonarAnalyzer ratcheted to blocking under `TreatWarningsAsErrors`, fix-or-silence, review-not-blind-fix.
- [[test-unit-registry]] вҖ” the area-prefixed Test Unit ID scheme and per-unit record behind the validation effort.
- [[definition-of-done]] вҖ” the six-criterion acceptance bar (TestSpec, tests, вүҘ80% coverage, edge cases, CI, evidence) each unit must clear.
- [[algorithm-validation-evidence]] вҖ” the templated per-unit `docs/Evidence/` artifact pattern (sources, worked-example dataset, deviations) behind the "Evidence documented" DoD criterion.
- [[global-alignment-needleman-wunsch]] вҖ” the canonical DP global-alignment algorithm (`GlobalAlign`): linear-gap recurrence, O(nm), traceback, affine-as-extension.
- [[multiple-sequence-alignment]] вҖ” aligning 3+ sequences (NP-complete): Seqeron's star (`MultipleAlign`), iterative-refinement (MUSCLE), and consistency (T-Coffee) aligners; sum-of-pairs + consensus.
- [[semi-global-alignment-fitting]] вҖ” ends-free "glocal" alignment: fitting/query-in-reference variant (Rosalind SIMS); NW recurrence with free reference end gaps, traceback from max of last row.
- [[alignment-statistics]] вҖ” post-alignment metrics: percent identity/similarity/gaps (EMBOSS/BLAST convention, denominator includes gaps, positive-score вҮ’ similar) and srspair three-line markup.
- [[coding-potential-hexamer-score]] вҖ” CPAT hexamer usage-bias coding-potential score: mean of ln(coding/noncoding) over in-frame hexamers (wordSize 6/step 3), sign convention, Вұ1 pseudo-scores; Fickett TESTCODE as not-implemented alternative.
- [[relative-synonymous-codon-usage]] вҖ” RSCU (Sharp & Li 1986) codon-usage-bias measure: n_iВ·x/ОЈx normalization, 1.0=no bias, bounded [0,n_i], ОЈ-over-family=n_i invariant; pooled counts, sense codons only; base anchor of the codon-usage family (vs CAI's max-normalization).
- [[codon-adaptation-index]] вҖ” CAI (Sharp & Li 1987) whole-gene codon-bias / expression-proxy score in [0,1]: geometric mean of relative adaptiveness w=f/max_synonym_f, low-value-sensitive, stop excluded; dual include/exclude-Met/Trp mode (exclude can give L=0вҶ’0); E. coli oracles; Seqeron deviation = 1e-6 zero-freq clamp (vs Sharp & Li's 0.5 reference-table pseudocount); one level above RSCU.
- [[rare-codon-analysis]] вҖ” codon-usage family's *localization/diagnostic* unit (`FindRareCodons` + cluster addendum): a codon is rare when its per-family frequency < threshold (default 0.15), reports position/AA/frequency (E. coli AGA 0.04/AGG 0.02/CGA 0.06/CUA 0.04); addendum adds two published cluster methods вҖ” Clarke & Clark 2008 %MinMax (18-codon window, signed %Min/%Max вҲҲ[вҲ’100,+100], rare clusters = negative peaks) and Chartier 2012 Sherlocc (7-codon window, вүҘ4 of 7 freqвүӨthreshold "slow" positions = a rare-codon cluster); single-codon-AA contributes 0 to %MinMax (no NaN), overlapping windows merged; feeds [[codon-optimization]]'s AvoidRareCodons; oracles вҲ’86.36/+100/+36.47% and 7Г—AGAвҶ’1 cluster; deviations None.
- [[codon-usage-comparison]] вҖ” codon-usage family's *raw* end (`CalculateCodonUsage` + `CompareCodonUsage`): unnormalized codon-count table (non-overlapping triplets, TвҶ’U, drop partial codon, Dictionary<codon,int>, ОЈcounts=total invariant) + Total Variation Distance similarity `1вҲ’ОЈ|fвӮҒвҲ’fвӮӮ|/2 вҲҲ [0,1]` between two codon-frequency distributions; proven identity 1.0 / symmetry / range [0,1] / disjointвҶ’0 / partial-overlap-exact properties; emptyвҶ’0, TVD-not-cosine deliberate metric choice; Kazusa-verified organism tables; shares RSCU's counting primitive, the distribution-comparison op none of the bias measures provide.
- [[codon-optimization]] вҖ” codon-usage family's CDS-*rewriting* operation (`OptimizeSequence`): synonymous substitution to improve host expression; five strategies (MaximizeCAI / BalancedOptimization 40-60% GC / HarmonizeExpression=Mignon 2018 / AvoidRareCodons / MinimizeSecondaryвҶ’ReduceSecondaryStructure); invariants protein-preservation + Met/AUG & Trp/UGG fixed + stop preserved + CAIвҲҲ(0,1]; RNA notation, TвҶ’U, trim-to-codon, 1e-6 CAI clamp; E. coli/yeast/human preferred-codon fixtures; actuator to RSCU/CAI/ENC's measurement.
- [[effective-number-of-codons]] вҖ” ENC/Nc (Wright 1990) reference-free whole-gene codon-bias measure in [20,61]: reciprocal of codon homozygosity FМӮ=(nВ·ОЈpВІвҲ’1)/(nвҲ’1) aggregated by degeneracy class (Nc=2+9/FМӮвӮӮ+1/FМӮвӮғ+5/FМӮвӮ„+3/FМӮвӮҶ); 20=max bias / 61=uniform, unbiasedвҶ’61, no-bias-simвҶ’40.5; Ile absentвҶ’(FМӮвӮӮ+FМӮвӮ„)/2 fallback, overshoot re-adjusts to 61; reference-free counterpart of CAI; one assumption = non-source-prescribed lower clamp at 20.
- [[repetitive-element-detection]] вҖ” repeats/tandem family anchor: tandem (head-to-tail, вүҘ2 copies, primitive unit), inverted (WВ·GВ·WМ„бҙҝ / zero-gap palindrome, IUPACpal), and RepeatMasker-class assignment; ClassifyRepeat = exact-substring containment, not Smith-Waterman homology.
- [[consensus-sequence]] вҖ” assembly CONSENSUS family anchor: column-wise majority/threshold consensus (Biopython dumb_consensus rule) вҖ” gaps skipped, emit iff unique max and freq вүҘ threshold else ambiguous, tieвҶ’ambiguous, all-gapвҶ’N no div-by-zero; N/0.5 parameterized defaults; shared with MSA consensus step.
- [[coverage-depth-calculation]] вҖ” assembly COVER family anchor: per-base sequencing depth over a reference (count of placed reads spanning each position, exact); average depth = ОЈdepth/G = Lander-Waterman C=LN/G, breadth = covered/G = 1вҲ’e^вҲ’c; boundary-clip + empty-input rules; ACGTTGCAATвҶ’1.5/1.0 oracle; Poisson (e^вҲ’c gap) as a property check only; read placement (minOverlap best-match) out of scope.
- [[de-bruijn-graph-assembly]] вҖ” assembly DBG family anchor: k-mer graph (distinct (k-1)-mers = nodes, each k-mer = one prefixвҶ’suffix edge, directed multigraph) + Eulerian-walk reconstruction (balanced/semi-balanced, Euler theorems 8.1/8.2, O(|E|) Hierholzer); spelling = path[0]+last char of each node; unique-walk oracles only (AAABBBA, a_longвҖҰ, to_everyвҖҰ k=4, ATGGCGTGCA); repeat/gap/error вҶ’ non-Eulerian; walk-selection unspecified so exact asserts on unique-walk inputs; OLC is the alternative formulation.
- [[overlap-layout-consensus-assembly]] вҖ” assembly OLC family anchor: the three-stage Overlap-Layout-Consensus paradigm (`FindAllOverlaps` + `AssembleOLC`) вҖ” overlap graph (read=node, edge on longest suffix-of-A/prefix-of-B вүҘ threshold), Layout = Hamiltonian path (NP-complete) + transitive reduction + non-branching contigs, Consensus = majority vote; GTACGTACGAT 12-edge / 5-overlap-tiling oracles; suffix-tree O(N+a) vs all-pairs O(NВІ); NP-complete/repeat-split/greedy-suboptimal failure modes; identity-1.0 + empty-input assumptions; Hamiltonian/overlap-graph alternative to DBG's Eulerian/k-mer formulation.
- [[contig-merge-overlap-collapse]] вҖ” assembly MERGE family anchor: suffixвҖ“prefix overlap collapse primitive (`MergeContigs`) behind greedy SCS and OLC layout; overlap = length-l suffix of X = prefix of Y, longest match reported, collapse keeps one copy so |merge|=|c1|+|c2|вҲ’l; overlap 0 / out-of-range вҶ’ plain concatenation; BAA+AABвҶ’BAAB, chainвҶ’AAABBBA oracles; caller-trusts-length + out-of-rangeвҶ’concat API-contract assumptions; overlap-based sibling of DBG's k-mer/Eulerian formulation.
