# Seqeron.Genomics — Current Limitations

**Library:** Seqeron.Genomics (mission-critical)   **Last reviewed:** 2026-06-25

This file lists **only what the library currently does NOT do** — genuine open limitations. Anything already
implemented (including opt-in alternatives, convention-parity modes, and resolved defects) is deliberately
**excluded**; those are recorded in the per-unit reports under `docs/Validation/reports/{UNIT}.md`, not here.

Every limitation below is `BY-DESIGN` and the unit is `✅ CLEAN` for its stated contract — these are honest scope
boundaries, never defects. They fall into three kinds:

- **Irreducible** — no algorithm or model can close it (physics / information theory).
- **Data-blocked** — needs a trained model / matrix / database that is gated, non-redistributable, or has never been measured / published. (Where the algorithm ships, the gated data is **caller-supplied** via a loader — the limitation is that nothing usable ships out-of-the-box.)
- **Scope** — a deliberate out-of-scope boundary; use the named reference tool, or supply the input yourself.

## 1. Irreducible (cannot be closed by any implementation)

| Unit | Not done | Why irreducible |
|------|----------|-----------------|
| PARSE-FASTQ-001 | Auto-disambiguation of Phred+33 vs Phred+64 when a read carries only overlap-range (ASCII 64–73) qualities. | The encodings overlap; the input is genuinely ambiguous (information-theoretic). Callers must decode with an explicit offset. |
| RNA-STRUCT-001 | Recovering tertiary-stabilised knots (e.g. BWYV / PDB 437D) as the MFE structure. | Not representable by *any* nearest-neighbour thermodynamic model — an energy-model floor, not an algorithm gap. |
| RNA-STRUCT-001 | Detecting pseudoknot classes outside the csr-PK grammar (kissing hairpins / loop–loop, triple-crossing / chained, non-canonical bulged helices). | The loop–loop interaction energy has never been experimentally measured (Sperschneider 2011 → heuristic estimate only); a faithful detector would need an unsourced energy constant. |

## 2. Data-blocked (needs a trained model / matrix / database that is gated, non-redistributable, or never measured)

| Unit | Not done | Blocking data |
|------|----------|---------------|
| ONCO-MHC-001 | The **embedded full 10-network ensemble weights** (~80 MB), and the *latest* NetMHCpan-4.1 ANN / MHCflurry presentation+processing models. Class I peptide-length coverage defaults to 8–11 (NetMHCpan admits 8–14). | A faithful C# port of MHCflurry's Class I **pan-allele binding-affinity** neural network now ships: the BLOSUM62 `left_pad_centered_right_pad` peptide encoding, the 37-residue allele **pseudosequence** table (bundled, Apache-2.0), the feed-forward forward pass (incl. MHCflurry's `with-skip-connections` topology), the `IC50 = 50000^(1−x)` transform, and the geometric-mean ensemble combiner — all oracle-verified against `mhcflurry` 2.1.5 (`models_class1_pan` 20200610) to <0.03% (`MhcflurryAffinityPredictor`, chained into `ClassifyBindingAffinity`). The Apache-2.0 trained **weights** total ~80 MB across the 10 ensemble members and are *not embedded* (a single member is embedded for the parity test); the full ensemble is loaded from a caller-supplied MHCflurry weight pack (`LoadWeightPack`). Matrix scoring (BIMAS/SMM) remains caller-supplied: BIMAS files are served only by a now-defunct CGI, the Parker 1994 table is paywalled, IEDB SMM matrices are non-commercial. |
| ONCO-IMMUNE-001 | The **CIBERSORT-LM22-identical** signature matrix specifically (the 547-gene × 22-cell-type LM22 is caller-supplied; exact-CIBERSORT parity is not claimed). The absolute-purity transform `cos(a+b·ESTIMATEScore)` is calibrated on Affymetrix / ABSOLUTE data (Yoshihara 2013), so it carries that platform caveat off-array. | LM22 is distributed by Stanford under a non-commercial **no-redistribution** licence and gated behind registration. A permissive signature matrix now ships out of the box: the **ABIS-Seq** matrix (Monaco et al., 2019, *Cell Reports*, CC BY 4.0; 1296 genes × 17 immune cell types) is bundled via `LoadBundledAbisSignatureMatrix()`, so immune deconvolution (the ν-SVR engine) works out-of-the-box — only LM22 itself remains caller-supplied. |
| META-BIN-001 | The **per-lineage-specific** CheckM marker refinement + the **reference genome tree** for lineage placement (the gated `checkm_data`). | Domain-level completeness/contamination now works out of the box: the Pfam (CC0) subsets of the GTDB bac120 (Bacteria, 6) and ar122 (Archaea, 35) universal single-copy marker sets ship bundled, feeding the CheckM formula. Only the per-lineage refinement + reference tree (`checkm_data`, a large gated trained DB) and the TIGRFAM-defined markers (CC BY-SA 4.0, not redistributable here) remain caller-supplied. |
| CHROM-CENT-001 | Separating **SF1 from SF2** (both dimeric with the identical A→B box pattern), and tagging SF3 arrays whose detected HOR period is not a multiple of five (e.g. the diverged-pentamer dodecameric DXZ1). | Suprachromosomal-family assignment now ships: `AssignSuprachromosomalFamily` classifies a detected α-satellite monomer/HOR against a bundled **CC0** Dfam reference (ALR/ALRa = A-type, ALRb = B-type with the CENP-B box) using HOR periodicity + A/B-box composition (Shepelev 2009; McNulty & Sullivan 2018, PMC6121732) — it resolves **SF3** (pentameric), **SF4** (monomeric A-type) and **SF5** (irregular A/B) and narrows dimeric arrays to **{SF1, SF2}**. The finer SF1-vs-SF2 / chromosome-specific HOR identity needs the **SF-resolved consensus monomer library** (J1/J2/D1/D2/W1–W5/M1/R1/R2), which is published only in supplementary matrices not machine-retrievable as FASTA and in third-party HMM repos (enigene/HumAS-HMMER, logsdon-lab) that ship **no licence** — not CC0/redistributable. Callers who hold an SF-resolved reference pass it via `AssignSuprachromosomalFamily(seq, reference)`. |
| DISORDER-REGION-001 | A calibrated per-residue / per-region disorder **confidence** value. | No disorder predictor publishes a calibrated confidence standard (the region boundaries themselves follow the validated TOP-IDP threshold). |

## 3. Scope boundaries (deliberate — use the named reference tool, or supply the input)

| Unit | Not done | Use instead |
|------|----------|-------------|
| PROBE-DESIGN-001 | The quantitative **MGB (minor-groove binder) ΔTm**, and dual-quencher labelling. | The MGB ΔTm model (Kutyavin 2000 / MGB-Eclipse) is empirical/proprietary with no published closed form — use a chemistry-specific tool; dual-quencher has no Tm impact. |
| MIRNA-TARGET-001 | The context++ feature `TA_3UTR` (transcriptome-wide site abundance) is now **computed from a caller-supplied 3'UTR set** via `ComputeTa3Utr` (Garcia 2011 / Agarwal 2015: `TA = log10(N)`, N = non-overlapping 8mer+7mer-m8+7mer-A1 site count of the seed across the 3'UTRs); only the 3'UTR set itself is caller-supplied (no default transcriptome is bundled — RefSeq is public domain but tens of MB). `PCT` (multi-species conservation) is computed from a caller-supplied alignment + tree (Friedman 2009 branch-length score → published logistic; the per-family sigmoid parameters are caller-supplied because TargetScan's compiled tables are citation-required and Friedman 2009 does not publish them as numbers). `SA` is computed; `SPS` / `Len_ORF` / `ORF8m` are caller-supplied. The score is a partial context++. | A 3'UTR set for `TA_3UTR` (then `ComputeTa3Utr` computes it); for `PCT`, supply the multi-species alignment + tree + the published per-family sigmoid parameters. |
| MIRNA-CLEAVAGE-001 | The exact miRBase-annotated **miRNA\*(3p) / star** boundaries on the *opposite* hairpin arm. The 5p mature Drosha/Dicer cut reproduces miRBase exactly; the 3p span is a **linear** 2-nt-3′-overhang offset of the 5p mature. Folding the precursor with the validated MFE engine (RNA-STRUCT-001) and placing the star on the base-paired arm was evaluated this session against three real hairpins with both arms annotated (hsa-mir-21 MI0000077, hsa-let-7a-1 MI0000060, hsa-mir-16-1 MI0000070) and does **not** recover the annotated 3p arms: the MFE-fold register implies a **1-nt** 3′ overhang at the Drosha (basal) end for mir-21/mir-16-1 — not the canonical 2-nt RNase III overhang — and for let-7a-1 the 5p 5′-start is **frayed (unpaired)** in the MFE structure, leaving the basal register undefined from pairing; the Dicer (apical) 3p 5′-start offset relative to the opposite-arm partner is likewise inconsistent (Δ = 0 for mir-21/mir-16-1, Δ = +2 for let-7a-1). miRBase mature boundaries encode the observed dominant **sequencing-read** cut sites (with processing heterogeneity), not a deterministic fold + fixed-overhang function, so no single source-backed rule recovers all three arms exactly. | The miRBase-annotated mature 3p coordinates (MIMAT IDs) for the opposite-arm star; or small-RNA-seq read pileups to call the dominant cut sites. |
| MIRNA-PRECURSOR-001 | The **read-stacking** (small-RNA-seq pileup) signal of miRDeep2 — the read-support component of its probabilistic precursor score. The *qualitative* model is published (Friedländer 2008, *Nat Biotechnol* 26:407; restated by miRDeep2, Friedländer 2012, *NAR* 40:37) as three biogenesis criteria — (a) mature read frequency > star > loop, (b) near-uniform Drosha/Dicer 5′ cut positions, (c) ~2-nt 3′ overhang between miRNA and miRNA\* — but the **closed-form log-odds score is NOT reproducible from the retrievable papers**: the NBT 2008 main text states only "a probabilistic model … log-odds score" (the equations live in the gated 2008 supplementary methods, which could not be retrieved), and NAR 2012 explicitly says "the core algorithm is identical to the first version" without restating the formula. The actual probability functions, priors, and constants exist only in (i) that gated supplement and (ii) the **GPL** miRDeep2 Perl source — reverse-engineering or translating the GPL code is out of bounds, so no faithfully-sourced exact scorer can be implemented. (A bundled **trained** structure/sequence-feature natural-vs-background classifier already ships: `ClassifyPreMiRna`, logistic regression over MFE/AMFE/MFEI/GC/%paired, trained from public-domain miRBase positives vs di-shuffled negatives — Bonnet 2004; held-out AUC = 1.0. NO GPL miRDeep2 code is used.) | miRDeep2 with the caller's small-RNA-seq reads (for the read-stacking signal); or the gated 2008 NBT supplementary methods for the closed-form log-odds. |
| PARSE-EMBL-001 | Fetching the **sequence** of a remote entry referenced in a location (the reference is parsed; only local parts are extracted). | Retrieve the remote accession from the source database. |

## How to read this

- This file lists capability/scope limitations only; pure **performance** optimisations (e.g. a genome-scale seeded index where an exhaustive scan already returns the same result) are not limitations and are not listed.
- For **research / pipeline** use, every row is a normal scope boundary of a from-first-principles library — the algorithm is validated correct for its stated contract.
- For **clinical / decision-grade** use, the §2 oncology rows mark layers that require an external validated predictor and clinical sign-off — the library computes the rule, not the trained model behind it.
- **Irreducible** rows can never be closed; **data-blocked** rows reopen only when the gated / unmeasured data becomes available (several already accept it via a caller-supplied loader); **scope** rows point to the reference tool, or accept the input from the caller.
- Each row traces to its per-unit validation report under `docs/Validation/reports/` (all `✅ CLEAN`). A unit that re-validated `PASS-WITH-NOTES` (🟡) but is **not** listed here carries only a by-design convention or an already-resolved doc/code note — recorded in its report, not a capability gap.
